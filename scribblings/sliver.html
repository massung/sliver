2 Quick Example

This is a brief example of slicing a string, which references the original instead of copying it. It’s possible to slice any integer-indexable sequence (technically anything that implements the sliceable? generic interface), which is any list?, vector?, string?, bytes?, flvector?, or fxvector?.

(define str "Hello, world!")
 
; slices can index using negative values as 'from end'
(define hello (slice str 0 5))
(define world (slice str -6 -1))
 
; slices can be indexed
(displayln (slice-ref world 2))
 
; slices are sequences and can use for loops and all sequence- functions
(for ([c hello])
  (display (char-upcase c)))
 
; slices can be displayed, which materializes them (copies from source)
(displayln hello)
(displayln world)

3 Introduction

Slicing is the act of taking a linear sequence (e.g., a list or vector) of elements and referencing a sub-sequence in it without copying it out. Fundamentally, a slice is just a source sequence and start/end indices. This package attempts to also make slices a bit easier to work with by optimizing list access and adding referencing, sequencing, and iteration functionality.

4 Slices

The slice function is used to create a new slice:

procedure

(slice sliceable [start end]) → slice?

  sliceable : sliceable?
  start : integer? = 0
  end : (or/c integer? #f) = #f
Creates and returns a new slice of sliceable.

The start and end arguments can be negative, in which case they are indexes from the end of the sequence.

If end is #f then the returned slice? will default to the slice-length of sliceable.

procedure

(slice? x) → boolean?

  x : any/c
Returns #t if x is a slice? structure. This is different from sliceable?, but all slices are also sliceable?.

procedure

(slice-of xs) → sliceable?

  xs : slice?
Returns the base sliceable? collection xs references. This is true even if xs is a slice? of a slice? of a slice? ....

(slice-of (slice (slice "Hello, world!" 0 5) 1 -1))  ; "Hello, world!"

procedure

(slice-start xs) → exact-nonnegative-integer?

  xs : slice?
Returns the aboslute start index of xs given its underlying sliceable? collection. As with slice-of, this is true regardless of how nested the slice? is.

(slice-start (slice (slice "Hello, world!" 2 8) 1 -1))  ; 3

procedure

(slice-end xs) → exact-nonnegative-integer?

  xs : slice?
Returns the aboslute end index of xs given its underlying sliceable? collection. As with slice-of, this is true regardless of how nested the slice? is.

(slice-end (slice (slice "Hello, world!" 2 8) 1 -1))  ; 7

5 Sliceable Generic Interface

The gen:sliceable interface should be implemented for any integer-indexable collection. It consists of the above four methods: slice-ref, slice-length, slice-range, and slice-materialize.

The built-in Racket types list?, vector?, string?, bytes?, flvector?, and fxvector? all implement these methods and so are both sliceable? and also slices themselves.

While the interface is fairly straight-forward. The only "tricky" bit are the definitions of slice-materialize and slice-copy. These are implemented the way they are so that the type of the underlying collection can determine how to materialize/copy a slice of itself, greatly simplifying the implementation of this package and allowing it to be extended to handle future types without knowing about them.

Generally speaking, slice-materialize for a base type can simply be the identity function, assuming the underlying type is immutable. Otherwise it could be implemented as a complete copy like so:

(define (slice-materialize xs)
  (slice-copy xs 0 (slice-length xs)))

The slice-copy method - given the source type, start and exclusive-end range - should return a copied collection of the same type, however that is best achieved for the type itself.

Here is the definition of the get:sliceable interface for vector?:

[vector?
 (define slice-ref vector-ref)
 (define slice-length vector-length)
 (define slice-range in-vector)
 (define slice-materialize identity)
 (define slice-copy vector-copy)]

procedure

(sliceable? x) → boolean?

  x : any/c
Returns #t if x implements the gen:sliceable interface.

procedure

(slice-ref sliceable i) → any/c

  sliceable : sliceable?
  i : exact-nonnegative-integer?
Indexes into the slice to return the value at the given index reference. This index is bounds checked and cannot exceed the length of the slice.

(slice-ref (slice "Hello, world!" 7 -1) 2)
#\r

procedure

(slice-length sliceable) → exact-nonnegative-integer?

  sliceable : sliceable?
Returns the length of the slice.

(slice-length (slice "Hello, world!" 7 -1))
5

procedure

(slice-range sliceable) → sequence?

  sliceable : sliceable?
Returns the slice as a sequence?. The slice struct implements the prop:sequence method, which calls this implicitly when needed for for loops.

(sequence-for-each display (slice "Hello, world!" 7 -1))
world
 
(for ([c (slice "Hello, world!" 7 -1)])
  (display c))
world

procedure

(slice-materialize sliceable) → sliceable?

  sliceable : sliceable?
A slice references the original source. Materializing a slice copies the values from the original collection out into a new collection of the same type. Ideally this shouldn’t need to be done very often.

(slice-materialize (slice "Hello, world!" 0 5))

The default implementation of this method for non-slice structures (e.g. lists, vectors, etc.) is simply the identity function.

procedure

(slice-copy sliceable start [end]) → sliceable?

  sliceable : sliceable?
  start : exact-nonnegative-integer?
  end : (or/c exact-nonnegative-integer? #f) = #f
Returns a new sliceable?, which is of the same type as sliceable, but a copy of the extracted range instead of a reference. If end is #f then the copy is from start to the end of the sliceable? collection.

It’s rare to have to call this function. It exists as an abstraction around slice-materialize, which should be used to extract and copy sub-ranges instead.
